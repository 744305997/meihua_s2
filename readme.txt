 四诊合参模拟器 (Four‑Diagnosis Simulator)

基于中医五行生克理论的**四诊合参辨证推演沙盘**。  
本项目是“脉象五行传变推演器”的二阶段升级版，将单一的脉诊推演扩展为望、闻、问、切四诊合参，支持流派自定义、多引擎对比、动态步骤干预和医案校准。

⚠️ **重要声明**：本工具仅供教学参考，所有推演结果均基于人为预设规则，**不代表真实生理状态，不可用于临床诊断**。

---

 🎯 核心功能

-   完整的四诊信息采集（望、闻、问、切），所有症状均可通过下拉菜单或勾选框输入。
-   四诊权重动态调节，可模拟“舍脉从症”或“舍症从脉”的不同辨证思路。
-   五行生克动态推演，支持单步演示和连续推演，实时可视化五脏能量变化。
-   辨证溯源：根据可配置的八纲、脏腑、六经规则，自动生成辨证结论及逻辑推导过程。
-   流派自定义：所有症状→五行映射、交互规则、辨证阈值均可通过图形界面编辑或通过 JSON 配置文件导入/导出。
-   多流派对比：同一组四诊信息，可同时用两个流派的配置并行推演，直观对比辨证差异。
-   动态步骤计划：可在任意步数手动注入自定义五行扰动、情志冲击或切换方剂。
-   医案校准器独立工具：根据医案阶段描述自动搜索最优引擎参数，生成可复现医案演变过程的配置包。
-   四象时钟手动覆写：支持自定义不同的时辰组合及其增益强度。
-   药剂干预（预设单味药及自定义方剂向量），贯穿推演全过程。

 📦 环境依赖与启动

1. **安装 Python 3.8+**，并确保 `pip` 可用。
2. **下载项目代码**至本地目录。
3. **双击 `启动.bat`** 即可自动创建虚拟环境、安装依赖并启动四诊合参模拟器（`app_stage2.py`）。  
   首次启动需要等待依赖安装完成，后续启动只需数秒。
4. **医案校准器**：双击 `启动校准器.bat` 单独启动。

也可在终端中手动执行：

```bash
 创建虚拟环境（可选）
python -m venv venv
venv\Scripts\activate

 安装依赖
pip install -r requirements.txt

 启动模拟器
streamlit run app_stage2.py

 启动校准器
streamlit run case_calibrator.py
```

浏览器会自动打开 `http://localhost:8501`。
